USAGE GUIDELINES:
- Database type: SQLite
- Use proper SQLite syntax
- Date format: YYYY-MM-DD
- Available regions: North, South, East, West
- Available products: Laptop, Phone, Tablet, Monitor, Keyboard, Mouse, Headphones, Speaker
- Revenue range: 100.00 to 5000.00 USD
- Use SUM(), AVG(), COUNT(), MAX(), MIN() for aggregations
- Use GROUP BY for categorical analysis
- Use ORDER BY for sorting results

SQLITE DATE FUNCTIONS:
- Use STRFTIME('%Y', order_date) to extract year from date
- Use STRFTIME('%m', order_date) to extract month from date  
- Use STRFTIME('%Y-%m', order_date) to get year-month
- Use STRFTIME('%Y-%m-%d', order_date) for full date formatting
- DO NOT use YEAR(), MONTH(), DATE() functions - they don't exist in SQLite

SAMPLE QUERIES:
- SELECT region, SUM(revenue) FROM orders GROUP BY region
- SELECT product, AVG(revenue) FROM orders GROUP BY product  
- SELECT COUNT(*) FROM customers WHERE region = 'North'
- SELECT STRFTIME('%Y', order_date) AS year, SUM(revenue) FROM orders GROUP BY STRFTIME('%Y', order_date)
- SELECT o.region, COUNT(o.id) as orders, COUNT(c.id) as customers FROM orders o LEFT JOIN customers c ON o.region = c.region GROUP BY o.region
